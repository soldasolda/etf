# ETF 적립 매수 시스템 컨셉 정리

## 1. 현재 방향

토스증권 Open API 승인을 기다리면서, 먼저 증권사와 독립적인 일일 리포트 및 승인 시스템을 만든다.

지금 단계의 목표는 실제 주문이 아니다. 매일 같은 기준으로 ETF 상태를 평가하고, 매수 제안을 만들고, 사용자의 승인/거절 기록을 남기는 것이다.

## 2. 시스템 정의

이 시스템은 시장을 예측해서 단기 수익을 노리는 자동매매 봇이 아니다.

목표는 다음과 같다.

- 장기 ETF 적립 투자를 지속 가능하게 만든다.
- 매달 투자할 돈을 조금 더 나은 구간에 나누어 투입한다.
- 감정 매매를 줄인다.
- 매수하지 않는 이유까지 설명한다.
- 토스 Open API가 준비되면 주문 연결부만 붙인다.

## 3. 브로커 구조

현재 코드는 브로커 연결부를 분리한다.

```text
app/brokers/
  base.py          공통 인터페이스
  sample_client.py API 없이 샘플 데이터 사용
  toss_client.py   토스 Open API 대기용 자리
  factory.py       BROKER 값으로 클라이언트 선택
```

초기 설정은 다음과 같다.

```env
BROKER=sample
```

토스 Open API 문서와 키를 받은 뒤에는 다음 구현을 추가한다.

```env
BROKER=toss
TOSS_APPKEY=<your_toss_app_key>
TOSS_SECRETKEY=<your_toss_secret_key>
```

## 4. 현재 구현 범위

- SQLite 저장소 초기화
- 샘플 일봉 데이터 생성
- 3개월 평균 계산
- 20일/60일 이동평균 계산
- 시장 라벨 계산
- 매수 점수 계산
- 일일 리포트 출력
- 매수 제안 생성
- 승인/거절 기록
- 승인 전 가격 재조회 및 변동률 검증 구조

## 5. 투자 운영 원칙

기본 적립금은 시장 참여를 유지하기 위한 자금이다.

전술 자금은 조건이 좋을 때 일부만 사용한다. 이 자금은 바닥을 맞히기 위한 돈이 아니라, 장기 평균 매입 단가를 개선하고 현금 방치를 줄이기 위한 완충 자금이다.

시스템은 다음 질문에 답해야 한다.

```text
오늘 살 것인가?
얼마나 살 것인가?
왜 사거나, 왜 기다리는가?
```

## 6. 점수 모델 초안

| 항목 | 의미 |
| --- | --- |
| 3개월 평균 대비 할인율 | 최근 평균보다 충분히 낮은지 |
| 20일선/60일선 추세 | 상승 추세가 유지되는지 |
| 최근 5거래일 수익률 | 단기 조정인지 급락인지 |
| 거래량 변화 | 하락/상승 강도 참고 |
| 현금 비중 | 전술 자금이 방치되는지 |

점수별 행동은 다음처럼 시작한다.

| 점수 | 행동 |
| ---: | --- |
| 0~39 | 관망 |
| 40~59 | 소액 분할 매수 제안 |
| 60~74 | 기본 전술 매수 제안 |
| 75~84 | 적극 분할 매수 제안 |
| 85~100 | 강한 매수 후보, 그래도 분할 집행 |

## 7. 시장 라벨

사용자에게 숫자만 보여주지 않고 현재 구간을 짧은 말로 설명한다.

- 상승 추세
- 상승 추세 중 단기 조정
- 횡보 구간
- 저가 매수 구간
- 하락 추세
- 공포 구간
- 과열 구간
- 관망 구간

라벨은 매수 조건의 대체물이 아니라 설명 장치다.

## 8. 승인 절차

주문 연결 전에도 승인 절차는 유지한다.

1. 일일 리포트가 매수 제안을 만든다.
2. 제안은 `pending` 상태로 저장된다.
3. 사용자가 승인 또는 거절한다.
4. 승인 직전 현재가를 다시 조회한다.
5. 제안 당시 가격과 차이가 크면 재승인을 요구한다.
6. 지금 버전은 주문을 보내지 않고 승인 기록만 남긴다.

## 9. 토스 API 문서 수령 후 확인할 것

- 인증 방식
- 시세 조회 API
- 국내 ETF 주문 가능 여부
- 주문 API
- 주문 조회/체결 조회 API
- 모의투자 또는 샌드박스 지원 여부
- 호출 제한
- IP 등록 또는 기기 인증 필요 여부
- 주문 취소/정정 가능 여부

## 10. 다음 개발 순서

1. 샘플 모드에서 리포트 문장과 점수 기준을 다듬는다.
2. 승인 기록과 일일 리포트 히스토리 조회 기능을 만든다.
3. 토스 Open API 문서를 받으면 `TossBrokerClient`에 시세 조회를 붙인다.
4. 시세 조회가 안정적이면 승인된 제안을 주문 API와 연결한다.
5. 주문번호와 체결 결과를 저장한다.
